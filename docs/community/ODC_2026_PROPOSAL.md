## Proposed Title

ローカル環境なしで完結する「最初のOSS PR」の設計

Designing a First OSS Pull Request That Needs No Local Environment

## Japanese Abstract

初めてOSSに参加する人は、修正内容よりも環境構築や対象イシューの選択で
止まりがちです。本セッションでは、韓国語話者向け日本語学習OSS
`AI Language Partner`で試している、GitHubのブラウザー編集だけで完結する
言語レビュー、ドキュメント、アクセシビリティ、テストフィクスチャの
タスク設計を紹介します。小さくても実際の利用者価値がある課題の切り方、
受け入れ条件、claim、レビュー、CI、貢献者へのフィードバックを、再利用
できるチェックリストとして整理します。プロジェクトはまだ初期段階で、
外部貢献者のマージ実績は3名です。完成した成功事例としてではなく、参加
障壁を下げながら品質と誠実な評価を守るための実践と失敗から得た知見を
共有します。

## English Abstract

First-time contributors often stop at environment setup or issue selection
before they can make a useful change. This session presents an early,
evidence-based experiment from AI Language Partner, a local-first Japanese
learning project for Korean speakers. Its language-review, documentation,
accessibility, and test-fixture tasks can be completed in GitHub's browser
editor while still delivering real user value. The talk covers task scope,
acceptance criteria, claiming, review, CI, and contributor feedback as a
reusable checklist. The project currently has three external contributors with
merged pull requests; this is a report on practical lessons and failures, not a
claim that the model has already scaled.

## Intended Takeaways

1. Convert a real product need into one reviewable first pull request.
2. Decide which tasks can honestly be completed without local setup.
3. Write acceptance criteria that help rather than script the contribution.
4. Keep CI, review, and contributor recognition auditable without gaming
   contribution metrics.
5. Reuse a checklist for documentation, localization, accessibility, and
   small test contributions.
